<known_gotchas>
- Lift-test calibration is saturation-only in scaled model space:
  `abs(saturation(x + delta_x) - saturation(x))`. Do not insert adstock unless
  a later fixture-backed contract proves Abacus is calibrating adstocked media.
- Inside `_time_series_mmm_model`, do not early-return after an invalid
  calibration log-density. Turing/DynamicPPL expects every model evaluation to
  execute the same set of `~` statements.
- In tests, prefer `import Turing` over `using Turing`; all test files are
  included into one shared `Main` namespace, so exported-name collisions can
  appear only in the full suite.
- Negative cost-per-target sigma is rejected at `CostPerTargetCalibrationRows`
  construction time, not `fit!` time.
- The calibration ledger row is still `scaffolded` pending Task 15-08; avoid
  release-facing claims that imply full calibration parity, panel support, VI
  support, or pipeline support.
</known_gotchas>
