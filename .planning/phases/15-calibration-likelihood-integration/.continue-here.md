  log-density helpers: `lift_test_estimated_lift_ad`,
  `lift_test_log_density`, and `lift_test_payload_log_density`.
- Task 15-05 wired the lift-test term into `_time_series_mmm_model` via
  `Turing.@addlogprob!`, using `centered_logistic_saturation` and the same
  sampled `lam` vector as the media saturation path. It converts valid-model
  domain failures during AD probes into `-Inf` rather than uncaught exceptions
  and avoids early returns so every Turing model evaluation touches the same
  `~` statements.
- Task 15-06 wired the cost-per-target soft-penalty term into
  `_time_series_mmm_model` via a second independent `Turing.@addlogprob!` call.
  It reuses `cost_per_target_total_penalty`; the payload fields are fixed
  caller-supplied `Float64` data resolved before sampling, so no new AD-safe
  helper was needed. Lift-test and cost-per-target terms are additive and
  independently optional.
- Task 15-07 added fixture-backed integration evidence. The main fixture
  exporter now emits
  `test/fixtures/abacus/calibration_integration_cases.jl`, generated from
  Abacus `scale_lift_measurements`, `add_saturation_observations`, and
  `add_cost_per_target_potentials`. `test/model/calibration.jl` verifies the
  resolved payloads and additive scalar log-density against the fixture, and
  `test/model/builder.jl` verifies that a conditioned `_time_series_mmm_model`
  logjoint differs from the uncalibrated model by exactly the fixture's
  combined term.

<known_gotchas>
- Lift-test calibration is saturation-only in scaled model space:
  `abs(saturation(x + delta_x) - saturation(x))`. Do not insert adstock unless
  a later fixture-backed contract proves Abacus is calibrating adstocked media.
- Inside `_time_series_mmm_model`, do not early-return after an invalid
  calibration log-density. Turing/DynamicPPL expects every model evaluation to
  execute the same set of `~` statements.
- In tests, prefer `import Turing` over `using Turing`; all test files are
  included into one shared `Main` namespace, so exported-name collisions can
  appear only in the full suite.
- Negative cost-per-target sigma is rejected at `CostPerTargetCalibrationRows`
  construction time, not `fit!` time.
- The calibration ledger row remains `scaffolded`; avoid release-facing claims
  that imply full calibration parity, panel support, VI support, pipeline/YAML
  support, non-logistic lift-test support, Dash/UI support, or AI-advisor
  support.
</known_gotchas>
