# External Code Review

## Findings

- high — `src/mmm/model.jl:171-191,201-238`, `src/mmm/panel.jl:107-136`, `src/model/results.jl:77-93,112-121`, `src/inference/results.jl:127-156,169-181` — Post-fit reporting and prediction rebuild runtime/spec state from the current mutable `model.config` instead of the stored fit artifact. A caller can change transform settings after `fit!` or `approximate_fit!` and still get `predict`, `model_results`, or `inference_results`, even though the posterior chain was drawn under the old configuration. This matters because predictive draws and grouped metadata can silently stop matching the fitted posterior parameterization; shortest fix is to treat the fitted artifact `spec`/`runtime` as canonical for post-fit surfaces, or reject config drift before generating results.

- high — `src/model/config.jl:28-49,199-206`, `src/model/types.jl:323-361`, `src/mmm/model.jl:242-250,293-294`, `src/mmm/panel.jl:142-147,166-167`, `test/model/config.jl:182-255` — `ModelConfig` validation never checks adstock or saturation semantics. Unsupported `type` values and malformed fields such as non-integer `l_max` or non-boolean `normalize` are accepted by config loading and can survive `build_model`, only failing later inside `_turing_runtime` / `_panel_turing_runtime`. This matters because the YAML/config contract is under-validated and users get late fit-time failures instead of immediate config errors; shortest fix is to add explicit adstock/saturation validators to `validate_model_config` and add negative tests for invalid types and field types.

- medium — `src/postmodel/replay.jl:157-165,252-266`, `src/mmm/model.jl:28-31,285-286`, `test/postmodel/contributions.jl:20-168`, `test/postmodel/response_curves.jl:21-189` — Deterministic replay silently substitutes `1.0` when `beta_media[i]` is missing. That fallback is only valid for Michaelis-Menten rows, where `beta_media` is intentionally omitted; for every other supported saturation path, a malformed or partially missing grouped posterior artifact still produces plausible-looking contributions and response curves instead of failing. This matters because downstream business outputs can be fabricated from corrupted grouped artifacts; shortest fix is to require `beta_media[i]` unless the replayed saturation path is Michaelis-Menten, and add a regression test that missing `beta_media` on a non-Michaelis-Menten artifact throws.

- medium — `src/distributions/special.jl:27-40`, `src/distributions/priors.jl:176-183`, `test/distributions/special.jl:41-66` — `SkewStudentT` only has a same-type positional constructor, and `instantiate_distribution` forwards raw config values directly. A mixed numeric config such as `nu = 7` with floating `mu`/`sigma`/`alpha` throws `MethodError` instead of constructing the distribution. This matters because integer-valued YAML or Dict configs are a normal public input path; shortest fix is to add a promoting outer constructor (or promote inside `instantiate_distribution`) and cover mixed int/float construction in tests.

- low — `src/distributions/special.jl:231-239,258-262`, `test/distributions/special.jl:20-21,60-61` — `Scaled` and `SkewStudentT` implement only scalar `rand(dist)` and miss standard `Distributions.jl` sampling forms such as `rand(rng, dist)`, `rand(dist, n)`, and `rand(rng, dist, n)`. Current tests only exercise scalar draws, so this interface gap is uncaught. This matters because reproducible and vectorized sampling fail for these public distribution types; shortest fix is to implement the standard RNG/array `rand` methods and add interface tests.

- low — `README.md:58-64`, `docs/src/index.md:76-80,108-150`, `src/model/builder.jl:297-302`, `src/Epsilon.jl` (export list), `src/transforms/convolution.jl:1-19` — The documented public surface does not match the exported one. `README` and the docs present `predict` as a supported entry point, but it is not exported and has no API doc block; the API page lists non-exported `ModelCoordinateMetadata`; exported `After`, `Before`, and `Overlap` are not documented individually. This matters because users and contributors cannot tell which names are intended stable entry points; shortest fix is to reconcile exports and docs in one direction: either export/document these names intentionally or remove the misleading public references.

## Residual Risks

- The time-series and panel feature matrix has broad happy-path coverage, but there is little adversarial testing around post-fit mutation, malformed grouped artifacts, or partially invalid config nodes, so contract drift can still hide behind passing integration tests.
- Post-modeling currently has useful bounded coverage for supported time-series bundles and VI, but robustness against externally loaded or hand-assembled `InferenceResults` remains thin.
- The custom distribution layer is small, but it is still only lightly exercised against the wider `Distributions.jl` interface, so additional compatibility gaps may exist outside the specific constructors and scalar sampling paths tested here.

## Conclusion

The repository is test-heavy at the layer level, but there are still a few contract-level gaps that are likely to matter in real use: mutable post-fit config can invalidate results silently, config parsing accepts transform states that are only rejected later, and parts of the public distribution/docs surface are narrower or less truthful than they appear.
